# Cursor AI 工作流命令

这个文件包含了用于管理 Cursor AI 助手工作流程的命令集合。这些命令可以帮助你更高效地与 Cursor AI 协作开发。

## 使用方法

在 Cursor 中，你可以使用这些命令来指导 AI 助手的行为。每个命令都有特定的用途，可以帮助你：
- 分析系统架构
- 调试问题
- 重构代码
- 修复错误

## 命令列表

### `/start`
**描述**: 读取项目结构，进入分析模式

**用途**: 当你想要 AI 全面理解项目架构时使用

**示例**: 
```
/start
```

### `/reset`
**描述**: 重置 Cursor 理解上下文（重新开始分析）

**用途**: 当 AI 的理解出现偏差，需要重新开始时使用

**示例**:
```
/reset
```

### `/analyze`
**描述**: 分析逻辑，不改代码

**用途**: 理解代码逻辑而不做修改

**示例**:
```
/analyze src/main.py
```

### `/findbug`
**描述**: 找出问题点但不修改

**用途**: 先分析问题，再决定如何修复

**示例**:
```
/findbug
```

### `/fixbug`
**描述**: 执行安全修复（小范围修改）

**用途**: 修复已分析出的问题，只修改受影响部分

**示例**:
```
/fixbug
```

### `/crash`
**描述**: 分析报错（Traceback）

**用途**: 当程序崩溃时，分析错误原因

**示例**:
```
/crash
[粘贴错误信息]
```

### `/refactor`
**描述**: 重建逻辑（当多次修复后混乱）

**用途**: 代码多次修改后逻辑混乱，需要重新梳理

**示例**:
```
/refactor src/event_manager.py
```

### `/output`
**描述**: 输出结果不对

**用途**: 当输出不符合预期时，分析输出生成过程

**示例**:
```
/output
```

### `/translate`
**描述**: 输出不是中文

**用途**: 确保所有输出都使用中文

**示例**:
```
/translate
```

### `/align`
**描述**: 字段不匹配（跨文件）

**用途**: 当不同文件间的数据字段不一致时使用

**示例**:
```
/align
```

### `/test`
**描述**: 让 Cursor 提示测试方法

**用途**: 需要了解如何测试某个功能时使用

**示例**:
```
/test
```

## 配置文件

命令定义存储在 `.cursor-commands.json` 文件中。你可以根据需要修改或添加新命令。

## 最佳实践

1. **先分析，后修复**: 使用 `/findbug` 和 `/analyze` 理解问题，再用 `/fixbug` 修复
2. **小范围修改**: 优先使用 `/fixbug` 而不是 `/refactor`，保持代码稳定性
3. **理解架构**: 使用 `/start` 确保 AI 理解整个系统后再进行修改
4. **调试流程**: `/crash` → `/findbug` → `/fixbug`

## 注意事项

- 这些命令是给 Cursor AI 的提示，不是程序代码
- 命令效果取决于 AI 的理解和执行
- 对于关键修改，建议先备份代码
